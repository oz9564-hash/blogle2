# AGENTS.md

이 문서는 이 레포에서 AI가 작업할 때 반드시 지켜야 할 기본 규칙입니다.

## 목적

- 여러 수강생이 AI를 통해 협업해도 결과물이 흔들리지 않게 합니다.
- SPA 구조와 헤더 유저 전환 구조를 유지합니다.
- AI가 임의 판단으로 공통 구조, 데이터 구조, 디자인 기준을 바꾸지 않게 합니다.
- 협업 운영 기준은 `docs/project-rules.md`에 두고, 이 문서는 AI 실행 규칙만 다룹니다.

## 기본값

### 문서 기준

- 협업, 브랜치, PR, 승인, 병합 기준은 `docs/project-rules.md`를 따릅니다.
- 역할, 더미 데이터, DB 구조 기준은 `docs/data-guide.md`를 따릅니다.
- 화면 디자인 기준은 `docs/design-system.md`를 따릅니다.
- Firebase 사용 범위, 환경변수, 배포 기준은 `docs/firebase-guide.md`를 따릅니다.

### 문서 확인 방식

- 모든 작업에서 모든 문서를 매번 읽지 않습니다.
- 작업 전에는 먼저 `README.md`의 문서 검토 순서와 이 문서의 실행 규칙을 확인합니다.
- 이후 작업 성격을 판단해 필요한 담당 문서만 추가로 확인합니다.
- 데이터 이름, 컬렉션, 필드, 상태값, 역할 값이 관련되면 `docs/data-guide.md`를 확인합니다.
- 화면 톤, 레이아웃, 컴포넌트, 반응형이 관련되면 `docs/design-system.md`를 확인합니다.
- 브랜치, PR, 병합, 승인, 작업 순서가 관련되면 `docs/project-rules.md`를 확인합니다.
- Firebase 연결, 환경변수, 배포, 보안 규칙이 관련되면 `docs/firebase-guide.md`를 확인합니다.

### AI 역할

- AI는 코드 수정, 문서 수정, 실행 확인, 빌드 확인, 커밋, PR 생성을 수행합니다.
- 수강생은 요구사항 결정, 기준 합의, 결과 검토, 승인 역할을 합니다.
- AI는 정해지지 않은 기준을 임의로 확정하지 않습니다.

### 작업 전 확인

- 수정 전 `README.md`, `AGENTS.md`, 작업 성격에 맞는 담당 문서를 먼저 확인합니다.
- 기존 파일 구조와 작성 방식을 먼저 확인합니다.
- 현재 Git 상태를 확인하고, 사용자나 팀원의 변경사항을 임의로 되돌리지 않습니다.
- 작업이 어떤 역할 화면에 영향을 주는지 먼저 구분합니다.

### 유지해야 할 구조

- 앱은 SPA 구조를 유지합니다.
- 헤더의 유저 전환 UI는 제거하지 않습니다.
- 현재 선택된 유저와 역할에 따라 화면이 달라지는 구조를 유지합니다.
- 역할 값은 임의로 늘리거나 이름을 바꾸지 않습니다.
- Firebase 로그인은 구현하지 않습니다.
- 실제 로그인 기능이 필요해 보여도 먼저 문서 기준을 확인합니다.

### 수정 기준

- 요청 범위 밖의 리팩터링은 하지 않습니다.
- 파일 삭제, 대규모 이동, 의존성 추가는 먼저 이유가 명확해야 합니다.
- 기존 컴포넌트, 스타일, 데이터 구조를 우선 재사용합니다.
- 새 화면이나 기능은 공통 디자인 시스템을 따라야 합니다.
- 화면마다 별도 디자인 톤을 만들지 않습니다.
- 문서에 정해진 기준과 다르게 구현해야 하면 문서 수정 여부를 먼저 검토합니다.

### 보고 기준

- 작업 후 변경한 파일과 확인한 명령을 짧게 공유합니다.
- 실패한 확인 작업이 있으면 숨기지 않고 말합니다.
- 선택지가 있으면 수정 전 상태와 수정 후 결과를 구분해서 설명합니다.

### 응답 기준

- 코딩 관련 답변에서 불필요하게 긴 설명을 하지 않습니다.
- 사용자가 요청하지 않은 코드는 본문에 길게 출력하지 않습니다.
- 선택이 필요한 경우 현재 상태와 바꾼 뒤 결과를 구분해서 설명합니다.
- 사용자 의견이 기술적으로 맞지 않으면 그대로 동의하지 않고 이유를 짧게 말합니다.

## 논의할 항목

- AI가 자동으로 수정해도 되는 범위
- AI가 반드시 확인받아야 하는 작업 범위
- 문서 기준과 코드가 충돌할 때 우선순위
- 테스트나 빌드 확인을 어디까지 필수로 할지
- AI 응답 방식과 코드 출력 허용 범위

## 최종 결정

- AI 자동 수정 허용 범위: 요청받은 코드와 문서 수정
- 사전 확인이 필요한 작업: 파일 삭제, 대규모 이동, 의존성 추가, 공통 구조 변경
- 유지해야 할 공통 구조: SPA, 헤더 유저 전환 UI, 역할 기반 화면 분기
- 필수 확인 명령: 변경 범위에 맞는 실행, 빌드, 린트 확인
- 커밋/PR 권한 기준: `docs/project-rules.md`를 따름
- AI 응답 방식: 짧게 결론 중심으로 보고하고, 요청 없는 코드는 본문에 길게 출력하지 않음
- 문서 확인 방식: 모든 문서를 매번 읽지 않고 작업 성격에 맞는 담당 문서만 추가 확인
