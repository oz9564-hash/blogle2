# 디자인 시스템

모든 화면이 같은 디자인 기준을 따르도록 정리한 최소 디자인 시스템입니다.

## 목적

- 수강생마다 다른 AI에게 화면을 맡겨도 결과물이 한 프로젝트처럼 보이게 합니다.
- 각 페이지가 임의의 색상, 여백, 버튼, 카드 스타일을 만들지 않게 합니다.
- 새 화면은 이 문서를 기준으로 만들고, 기준이 부족할 때만 문서를 먼저 수정합니다.
- 디자인 시스템은 개발을 막는 사전 승인 장치가 아니라, 화면의 결이 달라졌을 때 다시 맞추는 기준입니다.

## 기본값

### 디자인 방향

- 조용하고 실용적인 업무 도구 UI를 기본으로 합니다.
- 장식보다 정보 구조와 사용성을 우선합니다.
- 모바일에서도 깨지지 않게 만들되, 데스크톱 작업 화면을 함께 고려합니다.
- 같은 역할의 UI는 같은 모양과 동작을 유지합니다.

### 공통 레이아웃

- 헤더는 모든 화면에서 유지합니다.
- 유저 전환 UI는 항상 접근 가능해야 합니다.
- 본문 최대 너비는 기존 앱 기준을 따릅니다.
- 역할별 화면은 같은 간격, 카드, 표, 제목 체계를 사용합니다.

### 운영 방식

- 새 화면은 가능한 한 이 문서의 기본값을 참고해서 만듭니다.
- 기준이 부족하면 먼저 기능을 개발하고, 반복되는 패턴만 문서에 추가합니다.
- 이미 만든 화면이 서로 다른 톤으로 보이면 후속 정리 작업으로 맞춥니다.
- 모든 PR에서 디자인 시스템 완전 준수를 강제하지 않습니다.
- 단, 헤더, 유저 전환, 기본 색상, 버튼, 카드, 표 기준은 크게 벗어나지 않습니다.
- 디자인 정리가 필요하면 AI에게 "디자인 시스템 기준으로 화면 톤을 정리해줘"라고 요청합니다.

### 색상

- 메인 컬러는 현재 앱의 파란색 계열을 기본으로 합니다.
- 배경은 흰색 또는 아주 옅은 회색을 기본으로 합니다.
- 텍스트는 진한 회색, 보조 텍스트는 중간 회색을 사용합니다.
- 성공, 경고, 오류 색상은 의미가 있을 때만 사용합니다.
- 페이지별로 별도 컬러 테마를 만들지 않습니다.

### 타이포그래피

- 기본 폰트는 시스템 폰트를 사용합니다.
- 제목, 섹션 제목, 본문, 보조 텍스트 단계만 사용합니다.
- 과한 대형 제목이나 마케팅 페이지식 문구는 사용하지 않습니다.
- 버튼과 배지 안의 텍스트는 모바일에서도 잘리지 않게 합니다.

### 컴포넌트

- 버튼은 기본, 보조, 위험 버튼으로만 구분합니다.
- 입력창은 라벨, 안내 문구, 오류 문구 기준을 통일합니다.
- 카드는 반복 목록이나 독립 정보 묶음에만 사용합니다.
- 표는 데이터 비교와 관리 화면에 사용합니다.
- 아이콘은 필요할 때만 사용하고, 같은 의미에는 같은 아이콘을 사용합니다.

### 금지 기준

- 화면마다 다른 색상 테마를 만들지 않습니다.
- 페이지 전체를 카드처럼 감싸지 않습니다.
- 불필요한 장식용 그래픽이나 배경 효과를 넣지 않습니다.
- 기존 헤더와 유저 전환 UI를 디자인 이유로 제거하지 않습니다.
